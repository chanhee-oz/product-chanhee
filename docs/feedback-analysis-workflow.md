# 피드백 분석 워크플로우 설계

**풍수 진단 도구 — 피드백 수집/분석 체계**

## 1. 데이터 소스 현황

### 경로 1: 구글폼 피드백 (유저가 명시적으로 제출)
- type: 풍수 유형 (예: sunrise, mountain)
- match: 결과 일치 여부 (match / mismatch)
- reason: 불일치 사유 (more_energetic, more_calm, keywords_mismatch, tips_mismatch, unknown)
- inputs: 유저 입력 조합 (예: "east_mid_apartment_alone")
- imageSaved: 결과 이미지 저장 여부
- linkCopied: 결과 링크 복사 여부

### 경로 2: GA4 이벤트 로깅 (자동 수집)
- 각 스크린 도달률, 이탈 지점
- 상품 카드 클릭 (어떤 유형의 유저가 어떤 상품을 클릭했는지)
- 공유 버튼 클릭 (이미지 저장, 링크 복사, 카카오 공유 각각)
- 진단 완료까지 소요 시간

## 2. 분석 프레임워크

### 2-1. 고객 세그멘테이션 분포

inputs 필드를 파싱하면 방향_층수_집형태_동거인 조합을 분리할 수 있습니다.

- **집 유형별**: apartment/villa/house/officetel/oneroom 5개 그룹의 비율과 각 그룹별 만족도 차이
- **층수별, 방향별**: 각 선택지 분포. "모르겠어요" 선택 비율이 높으면 해당 질문 UX 개선 신호
- **동거인별**: 복수 선택 가능이므로 조합 빈도 확인
- **풍수 유형별**: 12개 유형 중 편중 여부. 특정 유형에 80% 몰리면 매핑 로직 밸런싱 필요

### 2-2. 만족도 분석

핵심 지표: **match rate** (일치 비율). 목표: 70% 이상.

- 전체 match vs mismatch 비율 트래킹
- mismatch 사유별 빈도 순위
- 유형별 match rate: 특정 유형의 mismatch가 유독 높으면 해당 유형의 키워드/요약/팁 수정
- inputs 조합별 match rate: "north_high" 조합의 mismatch가 높다면 TYPE_MAPPING 재조정

### 2-3. 상품 CTR/CVR 추적

상품 링크에 UTM 파라미터를 추가하여 추적:
`productUrl + '?utm_source=fengshui&utm_medium=result&utm_campaign=' + type.id`

GA4에서 추적 가능한 지표:
- fengshui 소스의 세션 수
- 상품 조회 수 (유형별)
- 구매 전환 수/율

### 2-4. 공유 행동 분석

- 이미지 저장율: 결과 도달 유저 중 이미지를 저장한 비율
- 링크 복사율: 결과 도달 유저 중 링크를 복사한 비율
- 공유 행동과 풍수 유형의 상관관계: 바이럴 포텐셜 높은 유형 식별
- 공유 행동과 match rate 상관관계: 결과가 맞다고 느낀 유저가 더 많이 공유하는지 확인

## 3. 자동화 워크플로우

### 3-1. 주간 자동 분석 리포트

리포트 포맷 (Slack에 주 1회 자동 게시):

```
[풍수 진단 주간 리포트] 2026-04-06 ~ 2026-04-12

핵심 숫자
- 총 진단 완료: 1,234건 (전주 대비 +15%)
- Match Rate: 72% (전주 71%)
- 이미지 저장율: 45% / 링크 복사율: 28%

유형 분포 (상위 5)
1. 타오르는 불꽃의 집 (fire) — 18%
2. 떠오르는 태양의 집 (sunrise) — 15%
...

주의 필요
- "새벽안개의 집" match rate 52% (전주 58%, 하락 추세)
- "잘 모르겠어요" 선택 비율: 층수 22%, 방향 18%

상품 성과 (UTM 기준)
- fengshui 소스 유입: 320 세션
- 상품 조회: 89건 / 구매 전환: 3건
```

### 3-2. 이상 징후 알림

다음 조건에서 Slack 알림 발송:
- match rate가 전주 대비 10%p 이상 하락
- 특정 유형의 mismatch가 50% 초과
- "잘 모르겠어요" 선택 비율이 30% 초과
- 일일 진단 완료 수가 전주 평균의 50% 미만 (유입 급감)
- 일일 진단 완료 수가 전주 평균의 300% 초과 (바이럴 발생)

### 3-3. 상품 추천 최적화 피드백 루프

1단계 (수동): 유형별 키워드/팁과 매칭되는 상품을 수동 큐레이션. 주 1회 CTR 데이터 기반 저성과 상품 교체.
2단계 (반자동): UTM 데이터 기반 유형별 상품 CTR 랭킹 자동 집계. 상위 유지, 하위 교체 후보 리스트 생성.
3단계 (자동화): 오늘의집 상품 API 연동 시, 유형별 키워드를 검색 쿼리로 자동 상품 풀 갱신.

## 4. BRD 문서화 구조

1. **개요**: 프로젝트 배경, 목적, 성공 지표(KPI)
2. **데이터 수집 요건**: 수집 항목, 수집 방법, 저장 위치, 보존 기간, 프라이버시
3. **분석 요건**: 세그멘테이션 기준, 핵심 지표 정의, 리포트 주기/수신자
4. **자동화 요건**: 자동 수집 주기, 리포트 생성/배포 로직, 알림 조건, 대시보드 명세
5. **개인화 연동 요건**: 오늘의집 플랫폼과의 데이터 연결 방식 (Phase별)
6. **기술 명세**: 사용 도구, API, 스키마, 접근 권한
7. **일정 및 단계**: Phase 1/2/3 마일스톤
8. **리스크 및 제약**: 프라이버시, 데이터 정확도, 샘플 사이즈 한계

## 5. 기술 구현: 자동 수집/분석 파이프라인

**구조**: 구글폼 응답 → 구글 시트 (자동 연결) → Apps Script → Slack 리포트

### 설정 단계

1. **구글폼-시트 연결** (1분): 구글폼 설정 → "응답" 탭 → "스프레드시트에서 보기"
2. **Slack webhook 생성** (5분): Slack API에서 incoming webhook URL 생성
3. **Apps Script 작성**: 시트에서 최근 7일 데이터 읽기 → match rate, 유형 분포, 공유율 계산 → Slack webhook으로 리포트 전송
4. **트리거 설정**: 매주 월요일 오전 9시 자동 실행

### 선택: Claude Code Scheduled Task 활용

매주 자동으로 구글 시트 데이터를 읽고 분석 리포트를 생성하는 것도 가능. Apps Script보다 분석 깊이를 더 줄 수 있음 (예: mismatch 패턴 해석, 개선 제안까지 포함).
