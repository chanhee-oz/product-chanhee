# 풍수지리 랜딩 페이지 디자인 스펙

## 개요

오늘의집 고객 대상 풍수지리 진단 인터랙티브 프로토타입. 고객이 재미있게 참여하는 과정에서 거주 지역, 집 형태, 환경 정보를 자연스럽게 수집하여 탐색/구매 경험 개선에 활용한다.

**핵심 전략**: 풍수지리라는 흥미 요소로 바이럴 참여를 유도하면서, 고객 거주 데이터를 자연스럽게 확보하는 투트랙 접근.

## 접근 방식

정적 싱글 페이지 (HTML + CSS + Vanilla JS). 프레임워크 없이 한 파일 세트로 완결. 배포 간단하고 빠른 로딩. 프로토타입에서 반응을 확인한 뒤 확장 여부를 결정한다.

## 1. 사용자 흐름

```
[인트로] → [Q1: 위치] → [Q2: 집 형태] → [Q3: 층수] → [Q4: 방향] → [Q5: 함께 사는 존재] → [로딩 연출] → [결과]
```

### 인트로 화면
- 타이틀: "우리 집에 숨겨진 기운, 알고 있나요?"
- 서브: "탭 몇 번이면 끝! 풍수로 읽어보는 우리 집 에너지"
- CTA: "우리 집 기운 읽어보기"

### 질문 설계

모든 질문은 선택형. 프리텍스트 타이핑 최소화. 선택 시 자동으로 다음 질문으로 슬라이드 전환. 상단 프로그레스 바로 진행 상태 표시 (1/5, 2/5...).

#### Q1. 풍수의 시작은 땅의 기운! 어디서 생활하고 있나요?

위치 정보를 부담 없이 제공할 수 있도록 두 가지 방식 제공:
- **"현재 위치로 찾기"** — GPS 기반, 탭 한 번으로 끝
- **"동네 이름으로 검색"** — "역삼동", "합정" 같이 동네 수준 입력, 자동완성 제공

안내 문구: "정확할수록 더 정밀한 풍수를 읽을 수 있어요! 하지만 대략적인 위치만으로도 충분해요"

내부적으로 동 단위 주소까지 확보하되, 사용자에게는 "서울 마포구 합정동 근처" 수준만 표시.

#### Q2. 어떤 공간이 당신을 품고 있나요?

선택지: 아파트 / 빌라·다세대 / 단독주택 / 오피스텔 / 원룸

#### Q3. 하늘과 얼마나 가까운 곳에서 살고 있나요?

선택지:
- 땅과 가까워요 (1~3층)
- 도시를 내려다봐요 (4~10층)
- 구름이 가까워요 (11~20층)
- 하늘 위에 살아요 (21층+)
- 잘 모르겠어요

#### Q4. 아침에 햇살이 어디서 들어오나요?

선택지:
- 동쪽 (아침 햇살 가득)
- 서쪽 (노을이 예뻐요)
- 남쪽 (하루종일 따뜻해요)
- 북쪽 (은은한 빛이 좋아요)
- 잘 모르겠어요

#### Q5. 이 공간의 에너지를 함께 만드는 존재는?

맥락 문구: "풍수에서는 공간의 기운이 함께 사는 존재들의 에너지와 섞인다고 해요"

선택지 (복수선택 가능):
- 나 홀로 충전 중
- 파트너와 둘이서
- 아이와 함께 북적북적
- 친구·룸메이트와 함께
- 반려동물이 함께!

### 로딩 연출 (약 2초)

단계별 텍스트 전환 애니메이션:
1. "기운을 읽는 중..."
2. "공간의 에너지를 분석하고 있어요..."
3. "거의 다 됐어요!"

## 2. 결과 화면

### 풍수 유형 체계 (12가지)

입력 조합(방향 x 집형태 x 층수)을 기반으로 12개 유형 중 하나에 매핑. 가족 구성과 지역은 에너지 수치에 가중치로 반영.

| 유형 이름 | 키워드 | 예시 조건 |
|----------|--------|----------|
| 떠오르는 태양의 집 | 활력, 시작, 성장 | 동향 + 중층 |
| 흐르는 물의 집 | 유연함, 풍요, 소통 | 저층 + 파트너 거주 |
| 단단한 산의 집 | 안정, 신뢰, 보호 | 단독주택 + 가족 |
| 고요한 달의 집 | 평온, 직관, 회복 | 북향 + 1인 거주 |
| 타오르는 불꽃의 집 | 열정, 창의, 에너지 | 남향 + 고층 |
| 깊은 숲의 집 | 치유, 성찰, 재생 | 단독주택 + 반려동물 |
| 바람길의 집 | 변화, 자유, 모험 | 고층 + 1인 거주 |
| 옥토의 집 | 풍요, 결실, 나눔 | 남향 + 가족 |
| 별빛의 집 | 영감, 꿈, 가능성 | 고층 + 오피스텔 |
| 새벽안개의 집 | 신비, 직관, 잠재력 | 북향 + 중층 |
| 돌다리의 집 | 견고, 인내, 성실 | 저층 + 아파트 |
| 봄바람의 집 | 새로움, 희망, 설렘 | 동향 + 룸메이트 |

### 결과 화면 구성

위에서 아래로 순서대로:

**1) 유형 타이틀**
- "당신의 집은... 떠오르는 태양의 집"
- 한 줄 요약: "새로운 시작의 에너지가 가득한 공간이에요!"

**2) 풍수 해설 ("어떻게 읽었냐면요...")**
- 사용자가 입력한 항목 각각에 대해 1~2줄의 풍수 근거 제공
- "풍수에서는 ~라고 해요", "전통적으로 ~로 봐요" 같은 인용 톤
- 전문적이되 어렵지 않게 — 친구가 풍수 좀 아는 것처럼 설명
- 각 항목 해설이 최종 유형 결과와 자연스럽게 연결

**3) 에너지 리딩**
- 활력 / 안정 / 풍요 3가지 축의 바 그래프
- 퍼센티지로 표시

**4) 개선 팁 ("이 공간의 기운을 더 살리려면?")**
- 1~2개의 구체적이지만 자연스러운 제안
- "초록 식물", "따뜻한 조명", "둥근 소품" 같은 일반적 표현
- 상품명이나 링크 없음
- "부족해서 바꿔라"가 아니라 "더 좋아질 수 있는 작은 변화" 톤

**5) 공유 영역**
- 결과 카드 저장하기 (이미지)
- 친구에게 공유하기 (URL)
- 나도 해보기 링크 복사

### 톤 원칙
- 절대 부정적 결과 없음 — 모든 유형은 고유한 강점 보유
- ~해요 체, 친근하고 가벼운 톤
- 이모지 적극 활용

### 공유 카드 (SNS용)

포함: 유형 이름 + 에너지 그래프 + "나도 해보기" URL
미포함: 주소, 층수, 가족 정보 등 개인정보 일체 없음

## 3. 기술 구조

### 파일 구성
- `index.html` — 전체 구조, 질문/결과 화면
- `style.css` — 스타일링, 애니메이션
- `app.js` — 질문 흐름 제어, 결과 매핑 로직, 공유 기능

### 질문 흐름 처리
- 각 질문을 카드 형태로 구성
- CSS 트랜지션으로 슬라이드 전환
- 상단 프로그레스 바 (1/5 ~ 5/5)
- 선택 시 자동 다음 질문 전환 (별도 "다음" 버튼 불필요)

### 위치 정보 (Q1)
- GPS: `navigator.geolocation` API → 카카오 역지오코딩으로 동 단위 변환
- 검색: 카카오 주소 검색 API로 동네 자동완성
- 내부적으로 동 단위 주소 확보, 사용자에게는 "OO구 OO동 근처" 수준만 표시

### 결과 매핑 로직
- 방향(5) x 층수(5) x 집형태(5)를 기반으로 12개 유형 중 하나에 매핑
- 가족 구성과 지역은 에너지 수치(활력/안정/풍요)에 가중치로 반영
- 매핑 테이블은 `app.js` 내 JSON 객체로 관리

### 공유 기능
- 이미지 저장: `html2canvas`로 결과 영역 캡처 → 다운로드
- 링크 공유: 결과 유형을 URL 파라미터에 인코딩 (예: `?type=sunrise`) → 공유받은 사람은 결과를 먼저 보고 "나도 해보기" 유도
- URL 복사: 클립보드 API

### 외부 의존성
- 카카오맵/주소 API (위치 기능)
- html2canvas (이미지 공유)
- 그 외 없음

## 4. 프로토타입 범위

### 포함 (MVP)
- 인트로 화면 + 5단계 질문 흐름 (선택형, 자동 전환)
- 위치: GPS 현재 위치 + 동네 검색
- 12가지 풍수 유형 결과
- 풍수 해설 (입력 항목별 근거 설명)
- 에너지 리딩 (활력/안정/풍요 바 그래프)
- 개선 팁 1~2개 (상품 링크 없이 자연스러운 제안)
- 결과 이미지 저장 + 링크 공유 + URL 복사
- 모바일 퍼스트 반응형 디자인

### 제외
- 지도에서 직접 찍기 (추후 추가)
- 카카오톡 공유 SDK 연동 (URL 복사로 대체)
- 이사 좋은 날 받기 기능
- 오늘의집 상품 직접 연결
- 백엔드/데이터 저장 (모든 로직 프론트에서 처리)
- 사용자 데이터 수집 서버

### 향후 확장 가능성
- 카카오 공유, 지도 선택, 상품 추천 연동
- 데이터 수집 서버 추가 → 고객 거주 정보 분석
- 이사 좋은 날, 계절별 풍수 운세 등 콘텐츠 확장
