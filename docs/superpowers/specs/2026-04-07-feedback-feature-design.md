# 피드백 수집 기능 디자인

## 목적
풍수 결과의 정확도 피드백을 수집하여 매핑 로직 개선 데이터를 확보한다. 만족도는 공유 행동(링크 복사, 이미지 저장)으로 간접 측정한다.

## 핵심 결정사항
- 피드백 유형: 결과 정확도 ("맞다/아니다")
- 저장 방식: Google Form pre-filled URL 백그라운드 전송 → Google Sheets 자동 적재
- 위치: 결과 화면 맨 아래 (공유 버튼 + 다시 해보기 아래)
- 불일치 시 추가 질문: 원인 기반 객관식 (클릭만, 텍스트 입력 없음)
- 모든 반응은 긍정적 프레이밍, ~해요체, 이모지 사용

## 플로우

### 1단계: 정확도 질문
결과 화면 하단, 다시 해보기 버튼 아래에 표시.

```
"이 결과, 우리 집이랑 어울려요?"

[😊 맞는 것 같아요!]  [🤔 좀 다른 느낌이에요]
```

### 2단계-A: "맞다" 경로
```
"역시! ✨ 친구 집 기운도 궁금하지 않아요?"
[🔗 친구한테 공유하기]
```
- 공유하기 버튼은 기존 링크 복사 기능과 동일하게 동작
- 공유하기를 클릭하지 않아도 OK — 반응 메시지가 표시된 시점에 피드백 데이터 백그라운드 전송

### 2단계-B: "아니다" 경로
```
"어떤 부분이 좀 다르게 느껴졌어요?"

[우리 집은 이것보다 더 활기찬 느낌이에요]
[우리 집은 이것보다 더 차분한 느낌이에요]
[키워드나 설명이 우리 집 분위기랑 달라요]
[팁이 우리 집 상황에 안 맞아요]
[그냥 느낌이 다른데 뭔지 모르겠어요]
```

### 3단계: 건설적 반응 (아니다 → 사유 선택 후)
| 선택 | 반응 |
|---|---|
| 더 활기찬 느낌 | "그렇군요! 🔥 활기찬 에너지가 넘치는 공간이라니, 실제로는 더 역동적인 기운이 흐르고 있나봐요. 소중한 의견 고마워요!" |
| 더 차분한 느낌 | "그렇군요! 🌙 고요한 에너지가 흐르는 공간이군요. 실제 기운은 숫자보다 깊을 수 있어요. 알려줘서 고마워요!" |
| 키워드/설명이 달라요 | "음, 공간의 기운은 사는 사람이 가장 잘 느끼는 법이죠! 🧭 더 정확한 풍수 리딩을 만드는 데 큰 도움이 돼요. 고마워요!" |
| 팁이 안 맞아요 | "맞아요, 같은 유형이라도 집마다 사정이 다르니까요! 🏠 더 맞춤형 팁을 만드는 데 참고할게요. 고마워요!" |
| 뭔지 모르겠어요 | "괜찮아요! 느낌이 다르다는 것 자체가 중요한 신호예요 ✨ 알려줘서 고마워요!" |

반응 표시 후 피드백 데이터 백그라운드 전송.

## 데이터 수집

### 전송 방식
Google Form pre-filled URL을 fetch로 백그라운드 전송. 고객은 폼을 직접 보지 않음. 응답은 Google Sheets에 자동 적재.

### 전송 데이터
| 필드 | 예시 값 | 용도 |
|---|---|---|
| 풍수 유형 | "sunrise" | 어떤 결과에서 피드백이 왔는지 |
| 맞다/아니다 | "match" / "mismatch" | 전체 정확도 비율 |
| 불일치 사유 | "more_energetic" (아니다일 때만) | 매핑 로직 개선 방향 |
| 입력 조합 | "east_mid_apartment_partner" | 어떤 조합에서 불일치가 나는지 |
| 이미지 저장 클릭 | yes / no | 저장 행동 |
| 링크 복사 클릭 | yes / no | 공유 행동 |
| 타임스탬프 | 자동 (Google Form) | 시간대별 패턴 |

### 수집하지 않는 것
- 위치 정보 (동네명, 좌표) — 프라이버시 원칙

### 불일치 사유 코드
| 선택지 텍스트 | 코드 |
|---|---|
| 더 활기찬 느낌이에요 | more_energetic |
| 더 차분한 느낌이에요 | more_calm |
| 키워드/설명이 달라요 | keywords_mismatch |
| 팁이 안 맞아요 | tips_mismatch |
| 뭔지 모르겠어요 | unknown |

## 구현 범위

### 변경 파일
- `index.html` — 피드백 섹션은 결과 화면에 동적 생성되므로 HTML 변경 최소
- `app.js` — 피드백 UI 렌더링, 클릭 핸들러, Google Form 전송 로직, 공유 버튼 클릭 추적 플래그
- `style.css` — 피드백 섹션 스타일 (기존 .result-section 패턴 활용)
- `data.js` — 변경 없음

### 새로 필요한 것
- Google Form 1개 생성 (위 필드에 맞는 항목 구성)
- Google Form의 pre-filled URL entry ID 확인

### 기존 패턴 준수
- 결과 화면은 `showResult()` 함수에서 innerHTML로 동적 생성 → 피드백 섹션도 동일 패턴
- 스타일은 기존 .result-section, .option-btn 패턴 재사용
- 화면 전환 없이 인라인으로 상태 변경 (classList toggle)

## 제약사항
- 텍스트 입력 필드 없음. 전부 클릭 기반
- 서버 추가 없음. Google Form pre-filled URL fetch만 사용
- 피드백은 1회만 가능 (제출 후 버튼 비활성화 또는 감사 메시지로 교체)
