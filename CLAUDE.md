# 풍수지리 랜딩 페이지 — Project Instructions

## 작업자 맥락
비개발자(CPO)가 바이브코딩으로 만드는 프로젝트. 코드 변경 시 무엇을 왜 바꾸는지, 어떤 영향이 있는지를 비개발자 눈높이로 같이 설명할 것. 대화는 반말/편한 존댓말 OK.

## 프로젝트 개요
오늘의집 고객 대상 인터랙티브 풍수 진단 도구. 풍수를 재미 요소로 활용해 주거 데이터(위치, 집 형태, 층수, 방향, 동거인)를 자연스럽게 수집하는 바이럴 프로토타입.

**목적:** 고객이 재미있게 참여하는 사이에 주거 관련 데이터를 확보 → 향후 탐색/추천 경험 개선에 활용 가능성 검증. 바이럴을 통한 자연 유입이 핵심.

## 기능 추가/수정 판단 기준
- 바이럴 임팩트가 있는가? (공유하고 싶어지는가)
- 데이터 수집에 도움이 되는가? (주거 맥락 파악에 기여하는가)
- 복잡도를 올리지 않는가? (Vanilla JS + 정적 호스팅 유지 가능한가)
→ 3개 중 2개 이상 충족하면 진행, 아니면 보류

## UX 원칙
- 결과는 항상 긍정적 프레이밍. 부정적 진단 없음
- 재미 > 정확성. 풍수 해석의 학술적 정확도보다 엔터테인먼트 가치 우선
- 공유 플로우가 핵심. 결과 화면 → 이미지 저장/링크 공유가 매끄러워야 함
- 상품 추천은 오늘의집 검색 URL(ohou.se/search/index?query=...)로 연결하거나 개별 상품 URL(store.ohou.se/goods/{id})로 연결

## 데이터/프라이버시
- 서버 전송 없이 클라이언트에서만 동작. 위치 정보 포함 모든 응답 데이터는 브라우저 내에서만 처리
- 향후 서버 연동 시에도 최소 수집 원칙 유지. 수집 항목 추가 전 반드시 논의

## 기술 스택
- Vanilla HTML/CSS/JS (빌드 스텝 없음)
- 외부 의존성: html2canvas (CDN), Nominatim API (역지오코딩)
- 정적 호스팅 (GitHub Pages)

## 파일 구조
- `index.html` — 메인 HTML (8개 스크린: intro, Q1-Q5, loading, result)
- `style.css` — 모바일 퍼스트 반응형 스타일
- `app.js` — 화면 전환, GPS/검색, 결과 렌더링, 공유 기능
- `data.js` — 12가지 풍수 유형 데이터, 매핑 로직, 해설

## 코딩 규칙
- 프레임워크 도입 금지. Vanilla JS 유지
- 모바일 퍼스트. max-width: 480px 기준, 데스크톱은 카드 레이아웃
- 한국어 ~해요체 (친근한 톤). 이모지 적극 활용
- CSS 컬러: 메인 액센트 #35C5F0 (오늘의집 시안)
- 새 기능 추가 시 기존 스크린 시스템(.screen + .active) 패턴 따를 것
